# Public dataset card and geographic coverage design

The dataset card will describe only public data concepts: polygons, Wikipedia and
Wikivoyage text, Wikidata facts, storage, languages, and geographic coverage.
Pipeline implementation terms such as “augmentation” will not appear in rendered
Markdown.

Coverage remains conditional on the dataset definition: every denominator polygon
already has an OSM `wikidata=*` tag. Wikipedia coverage is established through the
`polygon_articles` relation and non-blank canonical Wikipedia document text.
Combined text coverage additionally treats a polygon as covered when its Wikidata
QID matches a Wikivoyage document with non-blank full text. Each polygon is counted
once regardless of how many qualifying documents it has.

Continent statistics assign polygon centroids to Natural Earth Admin-0 countries and
their `CONTINENT` property. They report polygon counts, distinct Wikipedia and
Wikivoyage documents, Wikipedia-covered polygons, and combined-text-covered polygons.
Unassigned centroids remain visible as an explicit row rather than being discarded.

H3 cells crossing the antimeridian will be clipped into closed polygons inside the
world longitude bounds before rendering. This prevents Matplotlib from closing an
open fragment across the entire map. A new combined-text map is rendered first;
existing Wikipedia coverage, polygon-density, and polygon-centroid maps remain.

All statistics and images are deterministic functions of finalized Parquet files and
are regenerated by the existing publication workflow. Missing required columns or
geographic reference data fail with actionable errors rather than silently changing
the metric.
